# 🚂 Деплой на Railway.app — Пошаговая инструкция

## ✅ Что такое Railway?
Railway — бесплатный хостинг для Node.js приложений.
- **Не засыпает** (в отличие от Render)
- **Бесплатно** $5/месяц кредитов (хватает на небольшой проект)
- **Автодеплой** при каждом push в GitHub
- **Постоянный URL** вида `https://ваш-проект.up.railway.app`

---

## 📋 ШАГ 1 — Установить Git (если не установлен)

1. Откройте браузер, перейдите на https://git-scm.com/download/win
2. Скачайте и установите Git (нажимайте "Next" везде)
3. После установки откройте **новый** терминал (cmd) и проверьте:
   ```
   git --version
   ```
   Должно показать: `git version 2.xx.x`

---

## 📋 ШАГ 2 — Создать аккаунт на GitHub

1. Перейдите на https://github.com
2. Нажмите **Sign up** (Зарегистрироваться)
3. Введите email, пароль, имя пользователя
4. Подтвердите email

---

## 📋 ШАГ 3 — Создать репозиторий на GitHub

1. Войдите в GitHub
2. Нажмите кнопку **"+"** в правом верхнем углу → **"New repository"**
3. Заполните:
   - **Repository name**: `pvz-online` (или любое другое)
   - **Visibility**: `Public` (публичный — бесплатно)
   - **НЕ ставьте** галочку "Add a README file"
4. Нажмите **"Create repository"**
5. Скопируйте URL репозитория (вида `https://github.com/ВАШ_НИК/pvz-online.git`)

---

## 📋 ШАГ 4 — Загрузить код на GitHub

Откройте **cmd** (командную строку) и выполните команды по одной:

```cmd
cd "c:\Users\MyPc\Desktop\проекты на VS\Новая папка (5)"
```

```cmd
git init
```

```cmd
git add .
```

```cmd
git commit -m "Initial commit - PvZ Online"
```

```cmd
git branch -M main
```

```cmd
git remote add origin https://github.com/ВАШ_НИК/pvz-online.git
```
⚠️ Замените `ВАШ_НИК/pvz-online` на ваш реальный URL из шага 3!

```cmd
git push -u origin main
```

При первом push Git попросит войти в GitHub — введите логин и пароль.

> **Если просит токен вместо пароля:**
> 1. Перейдите на https://github.com/settings/tokens
> 2. Нажмите "Generate new token (classic)"
> 3. Поставьте галочку "repo"
> 4. Нажмите "Generate token"
> 5. Скопируйте токен и вставьте вместо пароля

---

## 📋 ШАГ 5 — Создать аккаунт на Railway

1. Перейдите на https://railway.app
2. Нажмите **"Start a New Project"** или **"Login"**
3. Выберите **"Login with GitHub"** — это важно! Так Railway получит доступ к вашим репозиториям
4. Разрешите доступ Railway к GitHub

---

## 📋 ШАГ 6 — Создать проект на Railway

1. После входа нажмите **"New Project"**
2. Выберите **"Deploy from GitHub repo"**
3. Найдите и выберите репозиторий `pvz-online`
4. Railway автоматически определит что это Node.js проект
5. Нажмите **"Deploy Now"**

---

## 📋 ШАГ 7 — Настроить переменные окружения (необязательно)

Railway автоматически установит `PORT`. Дополнительно можно добавить:

1. В проекте Railway нажмите на сервис
2. Перейдите во вкладку **"Variables"**
3. Добавьте если нужно:
   - `NODE_ENV` = `production`

---

## 📋 ШАГ 8 — Получить публичный URL

1. В Railway нажмите на ваш сервис
2. Перейдите во вкладку **"Settings"**
3. Найдите раздел **"Networking"** → **"Public Networking"**
4. Нажмите **"Generate Domain"**
5. Вы получите URL вида: `https://pvz-online-production.up.railway.app`

🎉 **Ваша игра теперь доступна в интернете!**

---

## 📋 ШАГ 9 — Проверить что всё работает

1. Откройте полученный URL в браузере
2. Должна открыться главная страница игры
3. Зарегистрируйтесь и войдите
4. Попробуйте найти игру

---

## 🔄 Как обновить игру после изменений

Каждый раз когда вы меняете код, нужно:

```cmd
cd "c:\Users\MyPc\Desktop\проекты на VS\Новая папка (5)"
git add .
git commit -m "Описание изменений"
git push
```

Railway автоматически задеплоит новую версию через 1-2 минуты!

## ⚠️ Важные замечания

### База данных (db.json)
Railway **не сохраняет файлы** между перезапусками! Это значит:
- При каждом деплое `db.json` сбрасывается
- Все пользователи и данные удаляются

**Решение:** Используйте Railway PostgreSQL или MongoDB Atlas (бесплатно).
Но для начала можно играть и так — просто знайте об этом.

### Бесплатный план Railway
- $5 кредитов в месяц
- Примерно 500 часов работы
- Для небольшого проекта хватает

## 🆘 Частые проблемы

### "Application failed to respond"
- Проверьте логи в Railway (вкладка "Deployments" → нажмите на деплой)
- Убедитесь что в package.json есть `"start": "node server.js"`

### "Port already in use"
- Это нормально на Railway — они сами назначают PORT
- В server.js уже стоит `process.env.PORT || 3000` ✅

### Игра не сохраняет данные
- Это из-за файловой системы Railway (см. выше про db.json)
- Для постоянного хранения нужна база данных
